Visual FoxPro 9.0 SP2
Cannot access file "file"; transaction is in progress. (Error 1106)






Visual FoxPro cannot access the table, index, or memo file because a transaction in progress is not yet complete.

FLOCK( ) and RLOCK( ) usually do not prevent other users from reading the locked information. This message, which scrolls from left to right to indicate that Visual FoxPro is still active, provides compatibility with the Novell transaction tracking system (TTS).
